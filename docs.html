Getting Installed

Install Meteor if not already installed... after meteor is installed via command line from a curl command on their website

$ meteor create ~/my_project_name $ cd ~/my_project_name $ meteor

Next - go to your server > main.js and remove all blaze related templating . erase entire file and replace with

import { Meteor } from 'meteor/meteor'; Meteor.startup(() => { });
  • install svelte for the front end with the "meteor npm install" $ meteor npm install --save svelte
  • add a couple packages to the project $ meteor add svelte:compiler rdb:svelte-meteor-data static-html
  • It must will look something like this.
  • go to your ".meteor" folder directory and find "packages". Go ahead and delete anything blaze related. specifically "blaze-html-templates" - remove this from the file completely. also add a new folder to your project named "imports". We will be using this to direct to our Svelte App Component
  • Edit your "src > main.html" to direct your html to js just like any app . react or other. head title project meteor svelte /title /head body div id="app" /div body

    Next - go to your client > main.js and replace with

    import { Meteor } from 'meteor/meteor'; import { App } from '../imports/ui/App.svelte'; Meteor.startup(() => { new App({ target: document.getElementById('app'), }); });

    • Adding Reactivity with Meteor/Mongo

    This section is largely in setting up a mongo collection and using useTracker to make it reactive.

  • -Create a new folder inside "imports" named "api" inside will be where js files will do mongo and collections are
  • Inside the newly created imports > api folder. lets create a file called whatever you want your mongo collection called. because that happens in this file. import { Mongo } from 'meteor/mongo' export const ContactForm = new Mongo.Collection("contactform")
  • Inside App.svelte add a script tag and insert the lines import { useTracker } from "meteor/rdb:svelte-meteor-data"; $: contactforms = useTracker(() => ContactForm.find({}).fetch());

    $: your_variable_name implies a REACTIVE VARIABLE in meteor. UseTracker works in tandem with $: to make synced and live reactivity.

    $: your_variable_name
